# Упражнения по юнитам — архитектура

Статус: проект (ничего из описанного ещё не реализовано в коде).
Дата: 2026-07-06.

## 1. Что мы имеем (анализ учебного материала)

### В Notion (видно интеграции)

```
Deutch (корневая страница)
├── Grammatik            — страницы по темам (Nomen, Verben, Adjektive, Artikel,
│                          Kasus, Zahlen/Datum/Zeit, Präpositionen)
├── wortschatz.csv       — база слов (208 шт., колонки Wort/Genus/Übersetzung/
│                          Lektion 1–12/Wortart/Beispiel/Österreichisch) → уже
│                          синхронизируется в public/data/words.json
├── Wortschatz - legacy  — старая база слов
├── Ressourcen
├── Austriazismen
└── A1-Prüfung — Roadmap & Checkliste
    ├── чек-листы грамматики (to-do) по областям
    ├── темы лексики, австрийский слой, экзаменационные умения
    └── таблица «Неделя курса → тема экзамена» (4 недели:
        W1 базовая грамматика · W2 падежи/Satzbau · W3 Perfekt/Schreiben ·
        W4 придаточные/Sprechen)
```

### Вне Notion

Детальные дневные планы («День 15 — Perfekt Teil 1» и т.д.) интеграции не видны —
они существуют отдельно. Структура дня (по прототипу):

1. **Грамматика** — короткая теория с примерами.
2. **🗣️ Говорение** — главное задание (произнести N предложений).
3. **📇 Слова** — список слов дня, ссылающийся на базу слов (с пометками 🔴).
4. **Упражнения** — 2–4 конкретных задания (дрилл форм, построение
   предложений, микро-дрилл рода).

Вывод: **юнит = день**, дни группируются в недели, недели — в план подготовки
к экзамену. Каждый день ссылается на подмножество слов из уже существующей
базы. Это и есть костяк модели данных.

## 2. Форматы упражнений (таксономия)

Разбор «Дня 15» даёт пять типов, которые покрывают и остальные дни плана:

| Тип | Пример из Дня 15 | Проверка |
|---|---|---|
| `transform` | «Образуйте Partizip II: machen → ?» | автоматическая (нормализация: trim/lowercase, допустимые варианты) |
| `gap-fill` | «Ich habe heute Deutsch ___ (lernen)» | автоматическая |
| `multiple-choice` | «reservieren → ge-reserviert или reserviert?» | автоматическая |
| `free-production` | «Постройте 6 предложений в Perfekt о своём дне» | MVP: самооценка + образцы ответов; этап 2: AI-проверка |
| `speaking` | «Расскажите вслух 5 предложений: Heute habe ich…» | самооценка по чек-листу (V2 haben? Partizip в конце?), таймер |
| `word-drill` | «5 минут der/die/das на существительных базы» | **уже реализовано** — WordsTrainer, нужен только фильтр по списку слов юнита |

Ключевое переиспользование: микро-дрилл рода/перевода — это существующий
тренажёр с параметром «пул слов = слова юнита». Новый код нужен только для
первых пяти типов, и они все — вариации «стимул → ответ → проверка», т.е.
ложатся в один discriminated union и один компонент-раннер.

## 3. Модель данных

```ts
// Юнит (день). Источник: Notion Units DB → sync-скрипт → public/data/units.json
interface Unit {
  id: string;            // notion page id
  day: number;           // 15
  week: number;          // 3
  title: string;         // "Perfekt Teil 1: haben + правильные глаголы"
  grammarSummary: TheoryBlock[]; // параграфы/списки/примеры из тела страницы
  wordIds: string[];     // ссылки на Word.id из words.json
}

// Упражнения юнита. Источник: AI-генерация → public/data/exercises/day-15.json
type Exercise =
  | { id: string; type: 'transform'; instructions: string;
      items: { stimulus: string; answer: string; accepted?: string[] }[] }
  | { id: string; type: 'gap-fill'; instructions: string;
      items: { textBefore: string; textAfter: string; hint: string; answer: string }[] }
  | { id: string; type: 'multiple-choice'; instructions: string;
      items: { prompt: string; options: string[]; correctIndex: number }[] }
  | { id: string; type: 'free-production'; instructions: string;
      minSentences: number; sampleAnswers: string[]; checklist: string[] }
  | { id: string; type: 'speaking'; instructions: string;
      prompts: string[]; checklist: string[]; minSeconds?: number }
  | { id: string; type: 'word-drill'; instructions: string;
      mode: 'gender' | 'translation'; wordIds: string[] };

// Прогресс — расширение существующего паттерна WordsProgressService:
// localStorage, ключ o-servus-exercise-progress,
// Record<exerciseId, { completedAt: string; score?: number }>
```

Принципы:
- немецкий контент в данных помечается на рендере `lang="de"` (уже принятый
  в проекте a11y-паттерн);
- `wordIds` ссылаются на реальные id страниц Notion из words.json — слова
  не дублируются в юнитах;
- один файл упражнений на юнит → ленивая загрузка, маленькие диффы при
  регенерации.

## 4. Где хранить (сравнение сред)

| Вариант | Плюсы | Минусы | Вердикт |
|---|---|---|---|
| **A. Статичные JSON в репо** (расширение текущего паттерна words.json) | нулевая инфраструктура; версионирование и ревью через git; офлайн; типы проверяются валидатором в CI | нет живого редактирования; прогресс только в localStorage | ✅ **MVP — да** |
| **B. Notion как CMS** (Units DB + sync-скрипт) | теория и метаданные юнитов редактируются там же, где весь учебный материал; консистентно с words-пайплайном | структурированные упражнения (варианты, ответы) в таблицах Notion неудобны — пришлось бы хранить JSON в текстовых полях | ✅ **частично: юниты/теория — да, упражнения — нет** |
| **C. Полноценный бэкенд** (Nest/Express + Postgres, либо BaaS типа Supabase) | авторизация, синхронизация прогресса между устройствами, рантайм-AI (генерация и проверка ответов), аудио для Hören | инфраструктура, хостинг, стоимость; для single-user MVP преждевременно | ⏳ **этап 2** |

### Рекомендация: гибрид A+B, бэкенд по критериям

```
Notion Units DB ──(sync-скрипт)──► public/data/units.json ─┐
wortschatz.csv ──(уже есть)─────► public/data/words.json ──┼─► Angular (learning)
теория+слова ──(AI-генерация)──► public/data/exercises/*.json ┘
```

- **Юниты**: завести в Notion базу «Units» (колонки: Day, Woche, Titel,
  Grammatik-Thema, Status; тело страницы — теория дня). Скрипт
  `sync-notion-units.mjs` по образцу существующего words-скрипта; маппер
  блоков уже есть (`mapBlockToContentBlock` в notion-mapper.ts).
- **Упражнения**: генерируются AI-скриптом (см. §5) и коммитятся как JSON.
  Ревью = git diff. Notion их не хранит.
- **Прогресс**: localStorage (паттерн WordsProgressService), ключи по id
  упражнения. Схема прогресса проектируется так, чтобы позже переехать на
  API без изменения компонентов (сервис-абстракция уже в стиле проекта).

### Когда действительно понадобится бэкенд (критерии для этапа 2)

Поднимать сервер стоит, когда наступит **хотя бы одно** из:
1. авторизация в Learning (в планах) и прогресс на нескольких устройствах;
2. AI-проверка свободных ответов в рантайме (ключ Anthropic нельзя держать
   в браузере — нужен прокси, хотя бы serverless-функция);
3. аудио-упражнения Hören (хранение/стриминг файлов);
4. живые данные из Notion без пересборки (тот самый прокси, под который уже
   спроектирован NotionService).

Форма: не обязательно «полноценный» сервер — Supabase (Postgres + auth + RLS)
или пара serverless-функций закрывают 1–2 без ops-нагрузки. Существующие
JSON-схемы станут контрактом API, компоненты не меняются.

## 5. AI-генерация упражнений (ресерч, без деталей реализации)

### Почему офлайн-генерация, а не рантайм

Тот же аргумент, что и с Notion-токеном: ключ Anthropic не может жить в
браузере. Рантайм-генерация требует бэкенда (этап 2). Для MVP — скрипт по
образцу `sync-notion-words.mjs`: запускается вручную, ключ в `.env`.

### Пайплайн

```
вход:  теория юнита (units.json) + слова юнита (words.json)
  │
  ▼
scripts/generate-exercises.mjs
  │  Claude API (@anthropic-ai/sdk — уже в devDependencies)
  │  • модель: claude-opus-4-8
  │  • structured outputs: output_config.format c JSON-схемой Exercise[]
  │    → гарантированно валидный JSON, невалидную структуру отдать не может
  │  • массовая генерация всех ~28 дней: Batch API (−50% к цене,
  │    обычно < 1 часа)
  ▼
public/data/exercises/day-N.json
  │
  ▼
валидатор (скрипт: wordIds существуют, ответы непустые, схема сходится)
  │
  ▼
git diff = человеческое ревью → commit
```

### Оценка стоимости (порядок величин)

Один юнит ≈ 2–4K токенов входа (теория + слова + схема) и 2–3K выхода.
На Opus 4.8 ($5/$25 за MTok) это ~$0.08–0.10 за юнит, **весь курс из ~28
дней — $2–3, с Batch API — $1–1.5**. Стоимость несущественна; можно
регенерировать сколько угодно раз.

### Контроль качества

- structured outputs снимает проблему «сломанного JSON» полностью;
- валидатор ловит ссылочные ошибки (несуществующие wordIds);
- языковую корректность (правильные Partizip-формы и т.п.) проверяет
  человек по git diff — на A1-объёмах это минуты;
- опционально: второй проход «модель проверяет упражнения модели»
  (self-review) перед записью — дёшево, но на MVP избыточно.

### Что оставить на этап 2 (рантайм-AI через бэкенд)

- проверка free-production ответов («мои 6 предложений в Perfekt») с
  фидбеком по ошибкам;
- адаптивная генерация (упражнения из слов, которые пользователь заваливает
  в тренажёре — данные для этого уже копятся в WordsProgressService);
- в перспективе — оценка говорения (Web Speech API → текст → проверка).

## 6. Прототип: «День 15» в этой модели

| Блок дня | Тип | Откуда данные |
|---|---|---|
| Теория Perfekt | `grammarSummary` юнита | Notion Units DB (тело страницы) |
| 🗣️ 5 предложений вслух | `speaking` | AI-генерация (промпты + чек-лист: «haben на 2-м месте?», «Partizip в конце?») |
| Partizip II от 8 глаголов | `transform` | AI-генерация из слов юнита (geben→gegeben, zeigen→gezeigt…) |
| 6 предложений о своём дне | `free-production` | AI-генерация (образцы + чек-лист), проверка — самооценка |
| Микро-дрилл рода 5 мин | `word-drill` | **готовый WordsTrainer** + `wordIds` юнита (die Stunde, …) |

UI: роут `/learning/units` = список недель/дней со статусом прохождения →
`/learning/units/:day` = страница юнита (теория → говорение → упражнения по
порядку, прогресс-бар). Слова дня — ссылка в таблицу слов с фильтром.

## 7. Порядок реализации (когда дойдём до кода)

1. Notion Units DB + `sync-notion-units.mjs` + модель Unit + страница юнита
   с теорией и списком слов (без упражнений).
2. Раннеры упражнений: `transform` / `gap-fill` / `multiple-choice`
   (один компонент, три режима) + `word-drill` через существующий тренажёр.
3. `speaking` и `free-production` с самооценкой + прогресс юнитов.
4. `generate-exercises.mjs` + валидатор + генерация Дня 15 как эталона.
5. (этап 2, по критериям §4) бэкенд: auth, серверный прогресс,
   AI-проверка свободных ответов.
